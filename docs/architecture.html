01

Executive Summary

The Mortgage Origination Platform processes residential mortgage applications through a multi-stage workflow—from applicant intake and credit scoring through underwriter decisioning and document verification. This document maps the current implementation and charts a future-state architecture with authentication, event-driven processing, external integrations, and enterprise-grade observability.

A

Current State

Monolithic three-tier architecture. Next.js SPA, Spring Boot REST API, PostgreSQL. Mock credit checks. No auth. Docker Compose deployment.

B

Future State

Event-driven microservices. OAuth2/OIDC auth, real credit bureau integration, object storage, notification service, API gateway, and full observability.

C

Migration Path

Incremental evolution. Auth and API gateway first, then extract services behind events. No big-bang rewrite—each step delivers standalone value.

Current Limitations

No Authentication

All endpoints are publicly accessible. No user identity, role-based access, or session management.

Mock Credit Scoring

Credit checks use a deterministic algorithm. No integration with Experian, Equifax, or TransUnion.

No File Storage

Document endpoints store metadata only. No actual file upload or object storage integration.

Synchronous Only

All processing is synchronous. No event bus, queues, or async workflows.

No Notifications

Status changes are not communicated. No email, SMS, or push notification capability.

No Observability

No structured logging, metrics, tracing, or health dashboard beyond basic /health endpoint.

An event-driven, microservices-oriented platform with authentication, real integrations, async processing, and enterprise observability. Each component is independently deployable and horizontally scalable.

Figure 2 — Future-state target architecture. Event-driven microservices with dedicated data stores, edge security, and full observability.

Future-State Component Detail

Detailed breakdown of each new and enhanced component in the target architecture.

Component Status Technology Responsibility
API Gateway New Kong / Spring Cloud Gateway TLS termination, rate limiting, request routing, CORS, API versioning
Identity Provider New Keycloak / Auth0 OAuth2/OIDC, RBAC (applicant, underwriter, admin), MFA, session management
Application Service Enhanced Spring Boot 3.2 Core CRUD, lifecycle management, financial calculations. Enhanced with event publishing and auth integration
Credit Service New Spring Boot 3.2 Real credit bureau integrations (Experian, Equifax, TransUnion). Multi-provider scoring with fallback
Document Service Enhanced Spring Boot 3.2 File upload to S3/Azure Blob, OCR extraction, virus scanning, verification workflow
Notification Service New Spring Boot 3.2 Event-driven email/SMS/push. Templates for each lifecycle event. SendGrid + Twilio
Workflow Engine New Camunda / custom state machine Orchestrates multi-step workflows, auto-assigns underwriters, tracks SLAs, escalation rules
Compliance Service New Spring Boot 3.2 AML/KYC checks, sanctions screening, fraud detection, regulatory rule enforcement
Valuation Service New Spring Boot 3.2 Automated Valuation Model (AVM), surveyor booking integration, property data enrichment
Reporting Service New Spring Boot 3.2 Real-time dashboards, portfolio analytics, FCA/PRA regulatory reporting
Event Bus New Apache Kafka Async event backbone. Topics for each domain event. Enables loose coupling between services
Redis Cache New Redis 7 Session storage, API response caching, rate limiting counters

07

Security & Access Control

Role-based access control with OAuth2/OIDC authentication, enforced at the API gateway and propagated via JWT tokens to all downstream services.

RolePermissionsPortal Access
Applicant Create, view, update, and submit own applications. Upload documents. View own audit trail. Applicant Portal, Mobile App
Underwriter View all submitted/under_review applications. Run credit checks. Make decisions. Add notes. Underwriter Portal
Admin Full access. User management. System configuration. Reporting. Audit log export. Admin Dashboard
System Service-to-service authentication via client credentials. Event bus producer/consumer. —
MFA Requirement: All underwriter and admin accounts require multi-factor authentication. Applicant accounts require MFA for document upload and application submission.
